You need two keys before your first call. See [Authentication](/authentication) for why.

## 1. Get an Alchemy key

WalletNest reads the chain with **your** Alchemy account, not its own — so your usage is yours, billed and metered on your own plan.

Sign up free at [dashboard.alchemy.com](https://dashboard.alchemy.com), create an app on **Ethereum Mainnet**, and copy the key.

```bash
export ALCHEMY_API_KEY=your_alchemy_key
```

## 2. Get a WalletNest key

Keys are self-serve. Nobody needs to issue you one.

<Steps>
  <Step title="Create an account">
    Sign up at [wallet-nest-nine.vercel.app/signup](https://wallet-nest-nine.vercel.app/signup).
  </Step>
  <Step title="Create a key">
    You land on your keys page. Give the key a name — "my bot", "staging" — and hit **Create key**.
  </Step>
  <Step title="Copy it immediately">
    The key is shown once. Copy it now.
  </Step>
</Steps>

```bash
export WALLETNEST_API_KEY=wnk_live_your_key_here
```

<Warning>
The key is displayed exactly once, at creation. We store only a SHA-256 hash of it, so we genuinely cannot show it to you again — not even if you ask. Lost it? Revoke it and create another.
</Warning>

You can create as many keys as you like, and **revoke any of them instantly** from the same page. Revocation takes effect on the very next request — there is no cache to wait out.

## 3. Call an endpoint

Send both keys. `Authorization` proves you may call the API; `X-Alchemy-Key` pays for the data.

<CodeGroup>

```bash cURL
curl https://wallet-nest-nine.vercel.app/api/v1/wallets/0xd8dA6BF26964aF9D7eEd9e03E53415D37aA96045 \
  -H "Authorization: Bearer $WALLETNEST_API_KEY" \
  -H "X-Alchemy-Key: $ALCHEMY_API_KEY"
```

```ts TypeScript
const res = await fetch(
  "https://wallet-nest-nine.vercel.app/api/v1/wallets/0xd8dA6BF26964aF9D7eEd9e03E53415D37aA96045",
  {
    headers: {
      Authorization: `Bearer ${process.env.WALLETNEST_API_KEY}`,
      "X-Alchemy-Key": process.env.ALCHEMY_API_KEY!,
    },
  }
);

if (!res.ok) {
  const { error } = await res.json();
  throw new Error(`${error.code}: ${error.message} (request ${error.requestId})`);
}

const wallet = await res.json();
```

```python Python
import os, requests

res = requests.get(
    "https://wallet-nest-nine.vercel.app/api/v1/wallets/0xd8dA6BF26964aF9D7eEd9e03E53415D37aA96045",
    headers={
        "Authorization": f"Bearer {os.environ['WALLETNEST_API_KEY']}",
        "X-Alchemy-Key": os.environ["ALCHEMY_API_KEY"],
    },
    timeout=30,
)
res.raise_for_status()
wallet = res.json()
```

## 4. Aggregate several wallets

`POST /v1/portfolio` fans out in parallel and rolls the results into totals.

```bash
curl https://wallet-nest-nine.vercel.app/api/v1/portfolio \
  -H "Authorization: Bearer $WALLETNEST_API_KEY" \
  -H "X-Alchemy-Key: $ALCHEMY_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{"addresses":["0xd8dA6BF26964aF9D7eEd9e03E53415D37aA96045"]}'
```

This endpoint is **partially fault-tolerant**. If one address fails upstream, you still get a `200`, and that wallet carries an `error` field while the others are intact. Totals only count wallets that loaded.

```ts
const { wallets, totalNativeBalance } = await res.json();

const failed = wallets.filter((w) => w.error);
if (failed.length) {
  console.warn(`${failed.length} wallet(s) failed`, failed.map((w) => w.address));
}
// totalNativeBalance already excludes the failed ones.
```

<Note>
A bad address is different from a failed one. Malformed input rejects the **whole** request with a `400`, because that's your bug, not the chain's.
</Note>

## Running locally

```bash
npm install
cp .env.example .env.local   # add ALCHEMY_API_KEY and WALLETNEST_API_KEYS
npm run dev
```

The base URL becomes `http://localhost:3000/api/v1`.
